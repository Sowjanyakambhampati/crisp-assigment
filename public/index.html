How it works

1
Enter your email below (or click a demo user). A magic link token is generated and its SHA-256 hash is stored in Postgres.
2
The raw token (never stored) is embedded in a URL. In production this URL is emailed; here we display it directly.
3
Clicking the link verifies the token by hashing it and looking up the hash. If valid & unexpired, a session is created.
